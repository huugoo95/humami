# Runbook: SEO Cycle

## Goal
Run repeatable SEO improvements without ad-hoc context switching.

## Weekly cycle (suggested)
1. Technical checks
   - indexability, canonical consistency, robots/sitemap sanity
2. On-page checks
   - title/description quality for priority pages
   - heading and internal link quality
3. Structured data checks
   - ensure valid schema where applicable
4. Content iteration
   - pick target pages/articles and define updates
5. Measurement snapshot
   - capture baseline metrics and compare next cycle

## Task format
Use:
`[SEO/Growth] objective | done-criteria | deadline`

## Deliverables per cycle
- Short report (what changed)
- Next-priority backlog (3-5 items)
- Risks/blockers

## Notes
- Keep SEO operational decisions in `ops/decisions.md`.
- Keep recurring findings in `ops/incidents.md` if they affect production behavior.
